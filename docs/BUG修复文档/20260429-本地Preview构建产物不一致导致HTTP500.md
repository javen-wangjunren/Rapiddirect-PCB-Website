title: 本地 Preview 构建产物不一致导致 HTTP 500
date: 2026-04-29
severity: medium
scope: 本地开发（astro preview / localhost）
---

## 1. 问题概述

本地访问后台编辑页面时出现：

- 浏览器提示 `HTTP ERROR 500`
- 终端日志出现 `ERR_MODULE_NOT_FOUND`，提示 `dist/server/chunks/*.mjs` 缺失

## 2. 影响范围

- `astro preview` 模式下，访问任意需要 SSR 的路由时可能 500（例如 `/ems/admin/pages/edit/`）
- 可能误判为“代码逻辑错误”，实际是构建产物与运行中的 server 进程不一致

## 3. 复现步骤（修复前）

1) 运行 `pnpm preview`
2) 在 preview 进程未停止的情况下再次运行 `pnpm build`
3) 打开任意 SSR 路由（例如 `/ems/admin/pages/edit/?slug=%2Fems%2F`）
4) 观察浏览器 500，终端报：`Cannot find module .../dist/server/chunks/<xxx>.mjs`

## 4. 根因分析

`astro preview` 运行时会从 `dist/` 目录动态加载 server chunks。
当 preview 进程运行中又触发 `astro build` 重新生成 `dist/` 时，
旧进程内存中仍引用旧的 chunk 文件名/路径，导致新 `dist/` 中找不到旧 chunk，进而触发 `ERR_MODULE_NOT_FOUND` 并返回 500。

## 5. 修复方案

- 先停止 `astro preview` 进程，再执行 `pnpm build`，最后重新启动 `pnpm preview`。
- 避免在同一时间让“正在运行的 preview server”与“正在写入 dist 的 build”并发。

## 6. 验证（手动）

1) 停止 preview（Ctrl+C）
2) 执行 `pnpm build`
3) 执行 `pnpm preview`
4) 打开 `/ems/admin/pages/edit/?slug=%2Fems%2F`
5) 确认不再出现 500，终端无 `ERR_MODULE_NOT_FOUND`

## 7. 自动化测试说明

该问题属于本地运行流程/产物一致性问题，仓库当前未引入 E2E 测试框架覆盖此类场景。
已通过手动步骤验证。
